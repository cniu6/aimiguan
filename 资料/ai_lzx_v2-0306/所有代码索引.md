## 2. `hfish/attack_log_sync.py` — HFish 日志同步

**路径**：`hfish/attack_log_sync.py`  
**功能**：从 HFish 蜜罐平台拉取攻击日志，写入本地 SQLite 数据库，支持持续同步与命令行参数。  
**依赖**：`requests`, `sqlite3`, `json`, `urllib3`  
**数据库**：`hfish/attack_logs.db`

### 数据库表结构

```sql
CREATE TABLE attack_logs (
    id                    INTEGER PRIMARY KEY AUTOINCREMENT,
    attack_ip             TEXT,        -- 攻击者 IP
    ip_location           TEXT,        -- IP 归属地
    client_id             TEXT,        -- 蜜罐节点 ID
    client_name           TEXT,        -- 蜜罐节点名称
    service_name          TEXT,        -- 被攻击的服务名
    service_port          TEXT,        -- 被攻击的服务端口
    threat_level          TEXT,        -- 威胁等级（高/中/低）
    create_time_str       TEXT,        -- 格式化时间字符串
    create_time_timestamp INTEGER      -- 时间戳（用于增量同步）
)
```

### 核心函数

| 函数名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| `load_config()` | - | dict | 读取 config.json |
| `timestamp_to_time(timestamp)` | int | str | 时间戳转可读时间（兼容毫秒级） |
| `get_attack_logs(start_time, end_time, host_port, api_key)` | int,int,str,str | list | 调用 HFish API 获取攻击日志 |
| `init_db()` | - | Connection | 初始化 SQLite 数据库，返回连接 |
| `get_last_timestamp(conn)` | Connection | int | 获取数据库最新记录时间戳（增量同步锚点） |
| `time_str_to_timestamp(time_str)` | str | int | 时间字符串转时间戳 |
| `save_to_db(conn, logs)` | Connection, list | int | 去重写入攻击日志，返回新增条数 |
| `main()` | - | - | 命令行入口，支持 --host/--key/--interval/--once |

### HFish API 调用细节

```
POST https://{host_port}/api/v1/attack/detail?api_key={api_key}
Body: {
    "start_time": <上次同步时间戳>,
    "end_time": 0,       // 0 表示当前时间
    "page_no": 1,
    "page_size": 1000,
    "intranet": -1,
    "threat_label": [],
    "client_id": [],
    "service_name": [],
    "info_confirm": "0"
}
```

### 去重逻辑

按 `(attack_ip, service_name, service_port, create_time_str)` 四元组去重，已存在则跳过插入。

### 命令行用法

```bash
python attack_log_sync.py --host 127.0.0.1:4433 --key <api_key> --interval 60
python attack_log_sync.py --once   # 仅同步一次
```

## 3. `nmap/network_scan.py` — 网络扫描与漏洞检测

**路径**：`nmap/network_scan.py`  
**功能**：使用 python-nmap 扫描指定 IP 网段，识别操作系统、开放端口、服务版本，执行针对性漏洞检测，持久化到 SQLite。  
**依赖**：`nmap (python-nmap)`, `sqlite3`, `json`, `os`, `time`  
**Nmap 可执行路径**：`nmap/Nmap/nmap.exe`（相对路径，Windows 特定）  
**数据库**：`nmap/network_scan.db`

### 数据库表结构

```sql
-- 扫描记录表
CREATE TABLE scans (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    scan_time    TEXT NOT NULL,      -- 扫描时间
    ip_ranges    TEXT,               -- 扫描的 IP 范围（逗号分隔）
    arguments    TEXT,               -- Nmap 扫描参数
    hosts_count  INTEGER DEFAULT 0   -- 本次扫描发现的主机数
);

-- 主机明细表（每次扫描独立记录）
CREATE TABLE hosts (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    scan_id      INTEGER NOT NULL,   -- 所属扫描 ID（外键→scans）
    ip           TEXT NOT NULL,
    mac_address  TEXT,
    vendor       TEXT,               -- 网卡厂商
    hostname     TEXT,
    state        TEXT,               -- up / down
    os_type      TEXT,               -- Nmap 识别的 OS 全名
    os_accuracy  TEXT,               -- OS 识别置信度
    os_tags      TEXT,               -- 自动生成的系统标签（逗号分隔）
    open_ports   TEXT,               -- 开放端口（逗号分隔）
    services     TEXT,               -- 服务信息（"port/name product version; ..." 格式）
    scan_time    TEXT,
    last_seen    TEXT,
    FOREIGN KEY (scan_id) REFERENCES scans(id)
);

-- 资产表（按 MAC 去重，保留最新状态）
CREATE TABLE assets (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    mac_address   TEXT UNIQUE,       -- MAC 地址（唯一键，无 MAC 时 NULL）
    current_ip    TEXT,              -- 当前 IP（可能变化）
    hostname      TEXT,
    vendor        TEXT,
    state         TEXT,
    os_type       TEXT,
    os_accuracy   TEXT,
    os_tags       TEXT,
    first_seen    TEXT NOT NULL,     -- 首次发现时间
    last_seen     TEXT NOT NULL,     -- 最后发现时间
    last_scan_id  INTEGER            -- 最后一次扫描 ID
);

-- 资产 IP 历史（追踪 IP 变动）
CREATE TABLE asset_ip_history (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    asset_id   INTEGER NOT NULL,
    ip         TEXT NOT NULL,
    scan_id    INTEGER,
    seen_time  TEXT NOT NULL,
    UNIQUE(asset_id, ip, scan_id),
    FOREIGN KEY (asset_id) REFERENCES assets(id)
);

-- 漏洞扫描结果表
CREATE TABLE vuln_scan_results (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    mac_address  TEXT NOT NULL,
    ip           TEXT NOT NULL,
    vuln_name    TEXT NOT NULL,      -- 漏洞脚本名称
    vuln_result  TEXT,               -- vulnerable / safe / error
    vuln_details TEXT,               -- 漏洞详情原始输出
    os_tags      TEXT,               -- 扫描时的系统标签
    scan_time    TEXT NOT NULL,
    UNIQUE(mac_address, vuln_name)   -- 同设备同漏洞只保留最新结果
);
```

### 核心函数

| 函数名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| `get_vuln_scripts_map()` | - | dict | 读取 config.json 中的漏洞脚本映射（优先），回退到内置默认配置 |
| `init_db()` | - | Connection | 创建所有表和索引，返回数据库连接 |
| `create_scan(conn, ip_ranges, arguments)` | - | int | 插入新扫描记录，返回 scan_id |
| `get_latest_scan_id(conn)` | Connection | int\|None | 获取最新 scan_id |
| `get_scan_list(conn, limit=20)` | Connection, int | list | 获取扫描历史列表 |
| `get_hosts_by_scan_id(conn, scan_id)` | - | list | 获取指定扫描的主机列表 |
| `scan_hosts(ip_range, arguments)` | str, str | PortScanner | 执行 Nmap 扫描，返回扫描器对象 |
| `parse_scan_results(nm)` | PortScanner | list | 解析 Nmap 结果，提取 IP/MAC/OS/端口/服务信息 |
| `auto_detect_os_tags(osmatch)` | dict | str | 从 osmatch 自动生成系统标签（win7/win10/linux 等），逗号分隔 |
| `get_vuln_scripts_for_os(os_tags)` | str | list | 根据系统标签获取对应漏洞脚本列表 |
| `get_vuln_status(conn, mac, vuln)` | - | str\|None | 查询某设备某漏洞的历史扫描结果 |
| `save_vuln_result(conn, mac, ip, vuln, result, details, tags)` | - | bool | 保存漏洞扫描结果（INSERT OR REPLACE） |
| `should_skip_vuln(conn, mac, vuln)` | - | bool | 若历史已确认 safe 则跳过（智能跳过优化） |
| `scan_vuln(nm, host_ip, vuln_script)` | - | (str, str) | 对单主机执行单漏洞扫描，返回 (result, details) |
| `run_vuln_scan(conn, hosts_data)` | - | dict | 批量漏洞扫描主流程，返回统计字典 |
| `upsert_asset(conn, scan_id, host, scan_time)` | - | - | 按 MAC 地址 upsert 资产，记录 IP 历史变动 |
| `save_to_db(conn, scan_id, hosts_data)` | - | int | 保存主机扫描结果到数据库，同时 upsert 资产 |
| `main(ip_ranges, scan_args, scan_interval)` | - | - | 主流程：扫描→解析→保存→漏洞扫描，0 表示只扫一次 |

### 系统标签识别规则（`auto_detect_os_tags`）

| 识别条件 | 生成标签 |
|----------|----------|
| Windows 10/11 | `windows, win10` |
| Windows 8/8.1 | `windows, win8` |
| Windows 7 / Vista | `windows, win7` |
| Windows XP/2000/2003 | `windows, winxp` |
| Windows Server | `windows, winserver` |
| Linux | `linux` |
| macOS/OS X | `macos` |
| BSD | `bsd` |

### 漏洞检测内置映射（`VULN_SCRIPTS_BY_TAG_DEFAULT`）

| 标签 | 漏洞脚本 | 说明 |
|------|----------|------|
| `windows` | smb-vuln-ms17-010, smb-vuln-ms08-067, smb-vuln-cve-2017-7494 | 通用 Windows 漏洞 |
| `win7` | smb-vuln-ms17-010, smb-vuln-ms08-067 | Win7 专项 |
| `win10` | smb-vuln-ms17-010 | Win10 专项（EternalBlue） |
| `linux` | ftp-anon, ssh-vuln-openssh | Linux 常见漏洞 |
| `ubuntu` | http-cakephp-version | Ubuntu 专项 |
| `debian` | ftp-anon | Debian 专项 |
| `centos` | ftp-anon | CentOS 专项 |

### 漏洞扫描智能跳过逻辑

- 若历史已确认 `safe` → **跳过**（减少重复扫描）
- 若历史已确认 `vulnerable` → **记录警告，不重复扫描**
- 若未扫描过 → **执行扫描**

### 启动流程

```
1. 读取 config.json
2. 初始化 HFish 数据库（非 reloader 进程）
3. 初始化 Nmap 数据库（非 reloader 进程）
4. 主进程（WERKZEUG_RUN_MAIN=true）启动后台线程：
   - HFish 同步线程（sync_enabled=true 时）
   - Nmap 定时扫描线程（scan_enabled=true 时）
5. 启动 Flask app（host:0.0.0.0, port:5000, debug:True）
```

## 6. 技术架构总结

### 技术栈

| 层次 | 技术 |
|------|------|
| 后端 | Python 3.8+ / Flask / SQLite3 |
| 扫描 | python-nmap / Nmap（Windows 可执行文件） |
| 数据同步 | requests（HFish REST API） |
| 前端 | Vue 3（CDN）/ Vuetify 3 / Chart.js |
| 图标 | Material Design Icons（mdi-*） |
| 字体 | Inter（Google Fonts） |
| 存储 | SQLite（两个库：attack_logs.db / network_scan.db） |
| 并发 | Python threading（daemon 线程） |

### 数据流

```
HFish 蜜罐平台
    → attack_log_sync.py（HTTPS API）
    → attack_logs.db（SQLite）
    → web_app.py /api/hfish/*
    → vuetify_hfish.html（Vue fetch）

nmap 扫描
    → network_scan.py（python-nmap）
    → network_scan.db（SQLite）
    → web_app.py /api/nmap/*
    → vuetify_nmap.html / vuetify_vuln.html（Vue fetch）
```

### 特色设计

1. **增量同步**：HFish 日志同步基于最后时间戳，避免重复拉取
2. **漏洞跳过优化**：已确认 safe 的漏洞不再重复扫描
3. **资产追踪**：按 MAC 地址识别设备，记录 IP 历史变动
4. **配置热重载**：后台线程每次循环重新读取 config.json，无需重启服务
5. **API 密钥保护**：GET /api/settings 不返回 api_key；POST 时若为空则保留原值
6. **线程安全**：扫描锁（is_scanning）防止 Nmap 并发扫描
7. **自动 DB 迁移**：`os_tags` 列不存在时自动 ALTER TABLE 添加
